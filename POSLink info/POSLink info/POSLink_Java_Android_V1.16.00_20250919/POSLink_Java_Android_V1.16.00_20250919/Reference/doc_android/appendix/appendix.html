Notes:

1. “\C” “\L” “\R” will not change line automatically, please add “\n” if you want to change to next line.

2. “\B” can be used along with “\L” or “\R” or “\C” with format like “\B\R”.

3. “\B” will automatically jump to next line if there are some words before “\B”.

4. “\B” will only bold the first line.

5. “\1”, “\2”, and “\3” will only apply the font size from where the control character was started and to the end of that specific lines text.

PrintData – Data format for printer

Control Character Description Comment
\L Use to indicate this is the left string(default)

Print text left alignment (default value)

\L will stop at next control character or \n or new line.

Example:

\LLeft or Left

Left
\R Use to indicate this is the right string

Print text right alignment will stop at next control character or \n or new line.

Example:

\RRight

Right
\C Use to indicate this is the central string

Print text center alignment

\C will stop at next control character or \n or new line.

Example:

\CCenter

Center
\B BOLD Not support
\I Inverted string

Print inverted string, the position of \I does NOT matter, it will stop at the next control character or \n or new line.

Example:

\Iwww or \L\Iwww or \I\Lwww

invert
\$Logo Use to print header image logo

It starts and terminates at new line, center align.

Limit:384*255 or less, unless will return PRINT LOGO ERROR.

Bit depth:1

\$LogoIMAGENAME or \$LogoIMAGENAME\n

First, zip picture (type: bmp), the file type in the zip package should be r, use command UPDATEIMAGE to upload image, then use command PRINT to print.

Example:

test.bmp properties:

dimensions:384*255

width:384 pixels

height:255 pixels

bit depth: 1

size: 12.0kb

attributes: A

command: \$Logotest or \$Logotest\n

print image:
\n Feed paper Carriage return, the same as enter
\$eSig Use to print signature

It starts and terminates at new line, center align.

Command: \$eSig\n or \$eSig
\$BARD,<code enum>,<size>,<data length>,<data> Use to print bar code.

It starts and terminates at new line, center align.

Fields are separated by comma (,).

Printing starts and terminates at new line (\n or next command) and bar code is center align.

Example:

1, Get the data by data length:

\$BARD,1,1,8,12345678

2, for EAN128, the first for data should be

[xxxx]. Like:

\$BARD,4,1,12,[3102]000035

3，for EN13, data length must be 12, like:

\$BARD,3,1,12,123456789012

Barcode example:

Enum1, CODE39:

\$BARD,1,1,6,123456

Enum 2, CODE 128:

\$BARD,2,1,8,12345678

Enum 3, EAN13 :

\$BARD,3,1,12,123456789012

Enum 4, EAN128:

\$BARD,4,1,12,[3102]000035

Enum 5, PDF417:

\$BARD,5,2,8,12345678

Enum 6, GRIDMATRIX:

\$BARD,6,8,8,12345678

Enum 7, QRCODE:

\$BARD,7,8,8,12345678
