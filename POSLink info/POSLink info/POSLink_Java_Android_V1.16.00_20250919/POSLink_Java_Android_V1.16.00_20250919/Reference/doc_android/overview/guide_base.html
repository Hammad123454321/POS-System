Base Usage Guide

For a successful transaction, the procedure should be: Setup CommSetting --> Setup Request --> Process POSLink --> Handle Process Result --> Get Response

// Setup comm setting. CommSetting commSetting = new CommSetting(); commSetting.setType(CommSetting.TCP); commSetting.setDestIP("192.168.0.1"); commSetting.setDestPort("10009"); // Create a POSLink object. In Android, it needs to pass context. PosLink posLink = new PosLink(); // Set the communication type. Currently support TCP, SSL, USB... For Android, it supports Bluetooth and more. posLink.SetCommSetting(commSetting); // Setup the Request. Can be PaymentRequest, ManageRequest, BatchRequest, ReportRequest ManageRequest manage = new ManageRequest(); // Set the TransType for the request first so that POS knows which part of data of request should be used. manage.TransType = manage.ParseTransType("INIT"); posLink.ManageRequest = manage; ProcessTransResult transResult = posLink.ProcessTrans(); // Get the response if transResult is OK ManageResponse response = poslink.ManageResponse;

A simple payment processing:

// Setup comm setting. CommSetting commSetting = new CommSetting(); commSetting.setType(CommSetting.USB); // Create a POSLink object. In Android, it needs to pass context. PosLink posLink = new PosLink(); // Set the communication type. Currently support TCP, SSL, USB... For Android, it supports Bluetooth and more. posLink.SetCommSetting(commSetting); // Setup the Request. Can be PaymentRequest, ManageRequest, BatchRequest, ReportRequest PaymentRequest pay = new PaymentRequest(); // Set the TenderType and TransType for the request first so that POS knows which part of data of request should be used. pay.TenderType = pay.ParseTenderType("CREDIT"); pay.TransType = pay.ParseTransType("SALE"); // Your unique ID for this transaction pay.ECRRefNum = "1"; //Optional fields. pay.Amount = "1100"; posLink.PaymentRequest = pay; ProcessTransResult transResult = posLink.ProcessTrans(); // Get the response if transResult is OK PaymentResponse response = poslink.PaymentResponse;

Data Attribute

In this doc, for each field, it has a data attribute to indicate the type of the field. For example, 'n' means number. See Data Attribute for detail.

Customize Form Management

Now some terminals support customizing the UI. If you have customize the form in the terminal, you can use POSLink to do some management.
For example, you can use RunForm command to control the terminal to show a form.
Also, you can set/get the variables for each form
For more detail, please check Java doc of package com.pax.poslink.customFormManage.
