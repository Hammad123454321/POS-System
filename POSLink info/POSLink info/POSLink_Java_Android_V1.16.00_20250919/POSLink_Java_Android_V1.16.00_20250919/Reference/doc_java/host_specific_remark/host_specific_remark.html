Host Specific Remarks

  • Bluefin Specifics

    • POS Echo Data

      • Terminal will map this as "&custom_data=<PAX>POS Echo Data</PAX>" in request message.

  • Heartland Portico Specifics

    • ECR Transaction ID

      • Terminal will map it to the <ClientTxnId> field in transaction request message. The <ClientTxnId> can only accept numeric value which means the current "ECR reference number" cannot not be used. Can be as long as 50 digits.

    • Adjustment

      • Heartland Portico Gateway only support transaction "edit", so POS needs to send total amount (base amount + tip amount) to the terminal when using HREF to do Tip Adjustment, since the terminal doesn't know the original sale amount when the adjustment is taken place on a different terminal, it will just send the amount whatever the POS system sends.

      When the adjustment is done on the same terminal and use OrigRefNum, the POS just needs to send the tip amount to the terminal to do Tip Adjustment.

  • TSYS Sierra Specifics

    • Gateway ID

      • Used to identify the Gateway, Aggregator, VAR, or other entity processing through the various TSYS Acquiring Solutions platforms. With max length as 10.

    • EMV Tag List

      • The following list of EMV tags are mandatory and must be sent on any transaction that contains Integrated Chip Card (ICC) EMV (group 3 version 55). If either or both of these tags are missing the transaction will be rejected.
      Tag Tag name Description
      DF78 Device Serial Number The manufacturer's unique serial number of the device that interacts with the chip card. (variable 1 - 20) It is for Contact and Contactless. The format is ASCII.
      DF79 Kernel Version Number The version number of the kernel used to process the chip data in the transaction. (variable 1 - 32) It is for Contact kernel only. The format is HEX.

  • TransIT Specifics

    • Transaction Adjustment

      • Transaction adjustment supports four types of amounts: base transaction amount, tip amount, surcharge fee, and tax amount. Transaction amount in the ECR-Terminal request message is base transaction amount + tax amount. So, the total amount when all values are set will be transaction amount+tip amount+surcharge fee. BroadPOS TransIT app supports transaction adjustment through local reference numer and host reference number. Using Local Reference Number: If user doesn't pass tip amount, tax amount, and surcharge fee, the BroadPOS app will set the missing values with original transactions corresponding amounts. Using Host Reference Number: If user doesn't pass tip amount, tax amount, and surcharge fee, the BroadPOS app will set these values to "0" in the host request message. Note: Transaction adjustment API is only supported by BroadPOS TransIT app.

  • First Data Rapid Connect Specifics

    • ECR Transaction ID

      • Terminal will map it to the field in transaction request message, numeric only with maximum length of 12.

    • eWIC Data

      • The fields in the PASSTHRUDATA for EBT eWIC transactions. This group can be repeated up to 50 times in a request. In case of a denial or exception this group can also be repeated up to 50 times in a response.

  • Shift4 Specifics

    • INCREMENTAL AUTH

      • If perform incremental auth by HRef, users need to set transaction amount and original transaction amount. If perform incremental auth by Reference Number, users just need to set transaction amount.

  • Vantiv Specifics

    • Customized Data

      • Terminal will map customized data to ‘G038-Customer Discretionary Data’. The max length of CUSTOMIZEDATA1 is 35. The max length of CUSTOMIZEDATA2 and CUSTOMIZEDATA3 is 20. It’s used in authorization request message to send up to 3 user defined data fields from the POS to the Front End.

  • BridgePay Specifics

    • Terminal will map CUSTOMIZEDATA1 to Custom Fields, CUSTOMIZEDATA2 to Memo, and CUSTOMIZEDATA3 to SoftwareVendor in the Host Request Message.

    Custom Fields supports up to an 800-character alphanumeric string. Contains plain text representations of custom fields.

    Example:

    <CustomFields>
    <Color>Red</Color>
    <Shape>Circle</Shape>
</CustomFields>

    Memo supports up to a 60-character alphanumeric string. The following characters are not allowed: “, *, &, <, >.

    SoftwareVendor supports up to 250 characters. May only contain the following characters: a-z, A-Z, 0-9, spaces and dashes.

    • Configurable Parameters Definition

      • Parameter Length (byte) Description R/W
      ServiceUser 150 The merchant user name for the service fee. W
      ServicePassword 50 The merchant password for the service fee. W
      MerchantAccountCode 10 The merchant account code for the current service user account.
      Note: if offline transactions exist in the database, this will not be allowed to change. This field is allowed to be changed when online transactions are in the database.       		R/W

  • JetPay Specifics

    • Customized Data

      • Terminal will map CustomizeData2 to UDField2, and CustomizeData3 to UDField3 in the Host Request Message. These fields can support up to 36 character alphanumeric strings. This is used in the authorization request message to send user defined fields from the POS to host.

    • Batch number (Batch ID)

      • Batch number (or BatchID) will not be supported on the Unattended version of this application, as the processor will automatically capture the transaction on the backend.
      The ECR may encounter the following as a result:
      1.Batch Number will not return in the transaction or report Response of a POSLink command.
      2.ECR cannot perform setvar or getvar on batchNum parameter without error.
      3.If Auto Clear Database is not enabled on the terminal, then ECR must call Batch Clear to prevent the database from reaching capacity.

  • Seamless Specifics

    • ECR Transaction ID

      • The terminal will map ECR Transaction ID to the host’s Idempotency Key in the payment request message to the host as an alphanumeric string up to 64 characters long. If ECR Transaction ID is not set in the request, the terminal will generate and the send the default one by using SN and timestamp. If host returns Idempotency Key, the terminal will map it to ECR Trans ID in the response.

  • Moneris Specifics

    • MSR TLV Data

      • The preferred language indicator is the last byte of the discretionary data before the end sentinel. 1 is English, 2 is French, 3 is Bilingual.

  • CardKnox Specifics

    • ECR Transaction ID

      • The terminal shall map "ECR Transaction ID" ("ECR Trans. ID" in low level specs) to the host’s "ECRTransID" field in the payment transaction request message. It accepts alphanumeric strings up to 64 characters long.
