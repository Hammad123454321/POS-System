Android Guidance

Dependency

See the README.txt under InstallPackage/libs/android/.

Full Integration

Full Integration APIs are under the package com.pax.poslink.fullIntegration.

PAX provides a flexible integration interface between the client App and PAX Android BroadPOS App on PAX Android devices. PAX provides a service running in the background. The service handles request from client App. Client App can customize its user interface according to the call back from the service. With this request and response, client App can get card information from the service to do transaction.

The service currently supports InputAccountWithEMV and other related process. With these, you can get card information from swipe card, dip card, and tap card to help you do a transaction. The interfaces are provided with Java SDK. The Java SDK is packed in aar or jar file. Client App can just make the aar file as dependence to integrate in the code.

Connect with AIDL or Connect with other communication type
The service app on Android supports connection based on AIDL. With AIDL, you can build your application in the same device with the service app.
If you use AIDL, you can also customize your user interface in the callback of each command.
If you don't customize your UI, it will use the built-in UI of the service app. These happens when you use other communication type or you don't implement the callback.

Customize Your UI
In full integration, you can customize your user interface in the steps of each API when you use Android with AIDL.
For example, if you use InputAccount.InputAccountCallback, you can show your UI when the call back triggered.
When onInputAccoutStart triggered, you can show a picture and message to tell the customer to swipe, insert or tap a card.

The order of the API call
1. Manual Entry: Just use InputAccount.
2. Swipe Card: Just use InputAccount.
3. Insert Card: InputAccount -> AuthorizeCard -> (Do online request by yourself) -> CompleteOnlineEMV -> RemoveCard
4. Tap Card: InputAccount -> GetPinBlock if need to input pin.

Initialize the whole library in Android

When using the library in Android, you have to initialize it in the start of the application.

Just call the

POSLinkAndroid.init(getApplicationContext());

For example:

public class MainApplication extends Application { @Override public void onCreate() { super.onCreate(); POSLinkAndroid.init(getApplicationContext()); } }

Using USB in Android.

Add the permission to AndroidManifest.xml:

<uses-feature android:name="android.hardware.usb.host"/>

See the demo for detail.

Peripheries

Currently some hardware is supported in Our Pax Devices, such as E500. In E500, you can use POSLinkPrinter, POSLinkScanner and POSLinkCashDrawer to control printer, scanner, cash drawer. Check the Java doc for details.

For A80 or other devices that support modem(dial), you can use modem API with com.pax.poslink.peripheries.POSLinkModem.
