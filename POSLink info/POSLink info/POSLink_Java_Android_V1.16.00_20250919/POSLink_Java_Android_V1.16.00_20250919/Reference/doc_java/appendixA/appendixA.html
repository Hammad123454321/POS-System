PassThru Data Definition

Define “PASSTHRUDATA” element to support Fleet, FSA, eWIC, Fleet, Level3 and Lodging data elements. This can be expended to other programs in the future.

Format as “PROGRAM<COLON>name1<COMMA>sub-value1<COMMA>subvaule2<COMMA>sub-value3<COMMA>sub-vlaue4<COMMA>... |name2<COMMA>sub-value1<COMMA>sub-value2<COMMA>sub-value3<COMMA>sub-value4<COMMA>…” .

Where

PROGRAM - is the name of the program. (“FLEET”, “FSA”,”Level3”, “eWIC”, “Lodging”, “Other”, etc…). The pos application will parse the PassThruData follow this value, each program have its own filed definition group pre-defined. When prepare the PassThruData, it must be suit what we pre-defined. Or the host application will parse error.

<COLON>‘:’ – program delimiter (Mandatory field)

<COMMA>‘,’ – name and sub-values or sub_valuesseparator (Optional field).

‘|’ – each group separator (conditional field)

“name1” – Mandatory field name

“sub_value1”/“sub_value2”/“sub_value3” /“sub_value4” values can be optional parameter (Optional field), these value can be set any values which needed in the Program requirement.
